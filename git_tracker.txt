Notes on realistic workflow and pacing:

- Ideal total commit count: 10–20 commits for an initial research repo of this size.
- Ideal total push count: 6–10 pushes — push often for shareable milestones, keep interim experimental commits local.
- Commits per day (realistic): 2–5 commits per active day, with intermittent days for analysis and writing.
- Grouping: Keep feature work isolated (preprocessing, feature engineering, modeling), push stable milestones; keep large data or heavy artifacts unpushed.

Replay guidance:

- Use `git init` and apply commits in the order above. For commits marked `NO`, keep those local or squash later when publishing a release branch.
- Ensure `data/raw` is large-file tracked externally (DVC or git-lfs) if necessary.

---

Recommended branch workflow:

- `main` (protected): release-ready documentation and stable scripts.
- `dev` (active): daily development work and integration.
- feature branches: `feature/preprocessing`, `feature/feature-engineering`, `feature/modeling`, `docs/spec`.
